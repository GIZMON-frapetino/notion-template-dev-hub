# 05_依頼受付->分類->担当割当 - Setup Guide

## 目的

依頼窓口を一本化し、分類・優先順位付け・担当割当を半自動化する。

## Prerequisites

- `Requests DB`（必須）
- `Tasks DB`（任意）
- `Projects DB`（任意）
- `Task Router Agent` を作成できる権限

## Step 1: Requests DB を準備

1. `db_structure.md` のプロパティを追加
2. 次のビューを作成:
   - 受信トレイ
   - 高優先度
   - 担当者別ボード
   - 完了アーカイブ

## Step 2: 判定ルールを用意（任意）

- `Routing Rules DB` にカテゴリ/担当者ルールを記載
- `SLA Rules DB` に優先度別期限を記載

## Step 3: Agent を作成

1. Agent 名を `Task Router Agent` に設定
2. `agent_instructions.md` を貼り付け
3. アクセス対象を `Requests DB` 中心に制限

## Step 4: テスト運用

1. テスト依頼を3-5件登録（バグ、問い合わせ、機能要望）
2. 手動実行で分類結果を確認
3. 誤判定があればルールを更新
4. 精度が安定したら定期実行またはイベント実行へ移行

## 安全運用ルール

- 自動分類後も `ステータス=未対応` で止める
- 担当者確定は人が最終確認
- 削除操作は行わず、ステータス更新で管理

## ロールバック手順

1. 誤分類レコードを特定
2. カテゴリ/優先度/担当者を手動で修正
3. 必要なら `テンプレ返信` を再生成
4. ルールDBを調整し再発防止

## FAQ

### Q1. カテゴリ判定がぶれる
- キーワードルールを明文化し、曖昧語を `その他` へ逃がす

### Q2. 担当者が偏る
- ルールに負荷分散条件（担当上限）を追加する

### Q3. クレジットを抑えたい
- 長文解析を避け、タイトル+冒頭テキスト中心で判定する
